Third-Party Banking Sandbox API

Production-style mock banking API suitable for white-label fintech integration in Nepal. Includes OAuth2 bearer authentication, scoped access, secure request header standards, account and transaction retrieval, and investment endpoints.

Token EndpointPOST /token
Token TypeBearer JWT
Subject ClaimPhone number
Default Expiry10080 minutes
Scopesaccount:read, transaction:read, transaction:write, stock:read
Protected endpoints require Authorization header: Bearer <access_token>

Signature Generation Example (Python)

import hashlib
import hmac
import json

secret = "bank-signing-secret"
request_id = "1c9fb11a-271f-4f6d-a2c2-7fce35d7b1a2"
payload = {
    "account_id": "b2a1c3d4-e5f6-7788-9900-aabbccddeeff",
    "amount": 1500.0,
    "currency": "NPR",
    "type": "DEBIT"
}

raw_body = json.dumps(payload, separators=(",", ":"), sort_keys=False).encode("utf-8")
message = request_id.encode("utf-8") + b"." + raw_body
signature = hmac.new(secret.encode("utf-8"), message, hashlib.sha256).hexdigest()

print("X-Request-ID:", request_id)
print("X-Bank-Signature:", signature)

Deployment Notes

  • Recommended base URL for local docs testing: http://localhost:8000
  • Protect secret material through environment variables only.
  • Use a reverse proxy and API gateway in non-local environments.
  • Persist idempotency keys with TTL for transaction create endpoints.
